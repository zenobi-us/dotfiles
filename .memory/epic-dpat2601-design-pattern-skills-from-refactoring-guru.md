# Design Pattern Skills from Refactoring.Guru

## Vision/Goal
Create a reusable skill pack that teaches and operationalizes software design patterns using Refactoring.Guru as the primary source. The output should be practical for coding-agent workflows: when to apply a pattern, tradeoffs, anti-pattern warnings, and implementation checklists.

## Success Criteria
- [ ] Define skill taxonomy aligned with pattern families (Creational, Structural, Behavioral).
- [ ] For each selected pattern, produce a skill artifact with intent, applicability signals, constraints, and usage steps.
- [ ] Include language-agnostic guidance plus implementation notes for the repo's common stacks.
- [ ] Add verification rubric so generated solutions can be checked for correct pattern application.
- [ ] Document legal/sourcing boundaries (derivative content limits, citation style, attribution).

1. `dpatp101` — Source audit, scope definition, and curriculum architecture
