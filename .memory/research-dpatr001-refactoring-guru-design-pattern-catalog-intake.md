# Research: Refactoring.Guru design-pattern catalog intake

## Research Questions
1. Does Refactoring.Guru provide a coherent taxonomy suitable for skill derivation?
2. What exact scope must be planned for authoring artifacts?
3. What per-pattern URLs and category descriptions should be preserved in planning docs?

## Summary
The source taxonomy is the classic GoF set of **22 patterns** grouped into **Creational (5), Structural (7), Behavioral (10)**. Planning scope is now explicitly fixed to one skill artifact per pattern URL.

## Findings

### Category descriptions (to retain verbatim in planning context)
- **Creational patterns**: provide object creation mechanisms that increase flexibility and reuse.
- **Structural patterns**: explain how to assemble objects/classes into larger structures while keeping them flexible and efficient.
- **Behavioral patterns**: cover algorithms and responsibility assignment/communication between objects.
